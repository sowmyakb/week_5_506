About Foodie

1. Project

Foodie — a Recipe Scaler and Meal Planner web app. Foodie is for people who want an easy way to plan their meals and adjust recipes without doing manual calculations. Users search recipes via the Edamam API, automatically scale ingredient quantities by serving size, and assign recipes to days in a simple weekly meal plan.

Custom variant: External-API consumer + CRUD. Users authenticate, search recipes from a third-party API (Edamam), persist saved recipes and meal plans in our own Postgres database, and view/edit/delete them through a Flask web UI.

4. Endpoints Planned

Draft — pending team review.

GET  /recipes/search?q=<query>          Search recipes via Edamam API
GET  /recipes/<id>                       View recipe details and nutrition
POST /plans                              Create a new weekly meal plan
GET  /plans                              List all of the user's meal plans
GET  /plans/<id>                         View one meal plan with its assigned recipes
POST /plans/<id>/items                   Add a recipe to a day/meal slot in a plan
POST /plans/<id>/items/<item_id>/scale   Scale servings for a planned recipe
POST /plans/<id>/delete                  Delete a meal plan

5. Web-Based Data Source

Primary API: Edamam Recipe Search API.

Specific endpoint: GET https://api.edamam.com/api/recipes/v2?type=public&q=<query>&app_id=<id>&app_key=<key>

Free-tier limits: approximately 10 monthly active users, limited requests per user per day, and restrictions on caching or storing recipe data — suitable for a demo/portfolio project rather than large-scale production.

Backup API: USDA FoodData Central API — free, ~1,000 requests/hour per IP, but only raw food/nutrition data (no recipes or images).
